## Navigation

### Community SPA Shell
- Three-route app: `/`, `/[communitySlug]`, `/help`
- Within `/[communitySlug]`, tab switching is instant client-side state — no URL change, no loading
- Tab state persists in localStorage (see `useTabPersistence`)

### Top Bar
- Fixed at top of the community SPA shell
- Left: Cader logo (50% opacity normally, 100% on hover) + vertical separator + community select component
  - Community select shows thumbnail + name of current community
  - Dropdown options: "+ Create new Community", "Explore Communities", separator, "My communities" + search icon (toggles to search input), list of user's communities (sorted by date, newest first)
  - Clicking a community navigates to that community
  - "+ Create new Community" opens existing creation modal
  - "Explore Communities" navigates to /explore route
- Right: Avatar (opens dropdown menu on click) + separate SearchIcon component
  - Avatar dropdown menu: Profile (opens profile modal), Settings (opens settings modal), separator, Help (navigates to /help), separator, Logout (red on hover, shows confirmation, clears auth token, redirects to "/")
  - SearchIcon component: clicking toggles search input visibility (appears to the right of the icon when active)

### Tab Navigation Bar
- Below the top bar on Desktop , fixed in bottom on mobile 
- Uses Frosted UI `Tabs` component — `Tabs.List` + `Tabs.Trigger` + `Tabs.Content`
- Tabs: About | Community | Classrooms | Members | Leaderboard | Analysis (owner only)
- Hidden when viewer is unauthenticated or not a member

### Back Navigation
- There is no back button inside the SPA — the browser back button returns to `/` .
- Classroom viewer opens within the Classrooms tab (no route change)
- Modals close via: X button, Escape key, clicking the backdrop
- the back button exist only in Classroom viewer

## Modals

### Pattern
- All modals use Frosted UI `Dialog` component
- All modals are centered dialogs on desktop, bottom dialogs on mobile.
- Modal backdrop: `bg-overlay/10 backdrop-blur-none`
- Opening animation: Framer Motion `opacity: 0→1, scale: 0.95→1, ,layer-blur 30px -> 0px duration: 150ms ease-out`
- Closing animation: no animation
- Frosted UI `Dialog` handles focus trap automatically

### Multi-step Modals
- Community creation and Onboarding modals are multi-step
- Steps shown as a progress indicator (e.g. "Step 1 of 2") in the modal header
- "Back" button on Step 2 returns to Step 1 without losing data
- "Continue" / "Create" / "Pay" are the primary actions — right-aligned, Frosted UI `Button variant="primary"`
- Secondary actions ("Cancel", "Skip") are left-aligned or `variant="ghost"` buttons

### Destructive Confirmations
- Pattern: user clicks destructive action → confirmation dialog overlays the current modal
- Confirmation requires typing the community name (for deletion) OR a plain "Confirm" button (for less severe actions)
- Destructive button: Frosted UI `Button variant="danger"`
- Cancel is always available and is the default focused element

## Forms

### Layout
- Single-column forms in modals
- Label above input (never inline placeholder as label)
- Use Frosted UI `TextInput` — it handles its own styling (border, background, focus ring)
- Helper text below input in `Text size="1" theme="muted"`
- Error message below input — replace helper text when error present, use `Text size="1" theme="danger"`

### Inputs (Frosted UI TextInput)

Frosted UI `TextInput` handles its own styling internally. Do NOT add Tailwind border/background
classes on top of it — use Frosted UI's prop API:

```tsx
import { TextInput, Text } from "@whop/react";

// Default
<TextInput placeholder="Community name" />

// With label and helper
<div className="flex flex-col gap-1.5">
  <Text size="2" weight="medium">Community Name</Text>
  <TextInput placeholder="my-community" />
  <Text size="1" theme="muted">Must be lowercase, 3–30 characters</Text>
</div>

// Error state (Frosted UI handles the visual)
<TextInput placeholder="my-community" error="This slug is already taken" />

// Disabled
<TextInput placeholder="..." disabled />
```

### Validation
- Validate on blur for most fields
- Validate in real time (debounced 300ms) for fields that need server-side checks (slug uniqueness)
- Validate on submit as final gate (server-side re-validation always happens)
- Show only the first error per field — not a list of all errors

### Submit Buttons
- Use Frosted UI `Button` — `variant="primary"` for main action, `variant="ghost"` for cancel
- Disabled + loading spinner while a mutation is in flight
- Loading state: Frosted UI `Button` handles loading state internally via `loading` prop
- Do not double-submit: disable the button immediately on first click

## Loading States

### Skeleton Screens
- Use Frosted UI `Skeleton` component — it handles the animation and dark surface color automatically
- Used for: feed posts loading, classroom grid loading, member list loading

```tsx
import { Skeleton } from "@whop/react";
import { Flex } from "@whop/react";

<Flex gap="3" className="w-full">
  <Skeleton size="8" />
  <Flex gap="2" className="flex-1">
    <Skeleton size="3" />
    <Skeleton size="2" className="w-24" />
  </Flex>
</Flex>
```

### Inline Spinners
- Used for: form submission, payment pending, single item loading
- Use Frosted UI `Spinner` or `Loader` component (check Frosted UI exports)
- Color: `text-mauve-8`

### Optimistic Updates
- **Not used for payment operations** — always webhook-gated, never optimistic
- Used for: post upvote count (show +1 immediately, revert on error), comment submit

## Empty States

### Pattern
- Centered in the content area
- Frosted UI `Skeleton` or `Callout` background (muted)
- Icon (Frosted UI or Lucide, 32–40px, `text-mauve-8`)
- Frosted UI `Heading` (size 5 or 6) for the heading
- Frosted UI `Text` (size 2, `theme="muted"`) for sub-text
- Frosted UI `Button` CTA (optional, only if there's a direct action)

### New Community (No Posts)
- Community feed: illustration + "Be the first to post" + post composer CTA
- Classrooms: grid shows only the "+" add card (owner) or "No classrooms yet" (member)
- Members tab: map shows, list shows "No members yet"
- Leaderboard: "Earn points by posting and getting upvotes"

## Error Handling

### Toast Notifications
- Library: Frosted UI `Toaster`
- Frosted UI `Toaster` handles types automatically via `variant`: `"success"`, `"error"`, `"warning"`, `"info"`
- Position: bottom-center on desktop, top-center on mobile
- Duration: 3.5 seconds (auto-dismiss)

```tsx
import { use toast } from "@whop/react";

// In a component:
const { toast } = useToast();
toast({ title: "Community created", variant: "success" });
toast({ title: "Something went wrong", description: "Please try again", variant: "error" });
```

### Inline Errors
- Form field errors: Frosted UI `TextInput error="..."` prop — renders below the field (see Forms section)
- Page-level errors (e.g. community not found): full-page centered error state with a back-to-home link
- Webhook pending error (payment not confirmed): shown inside the onboarding modal (not a toast)

### Never Show
- Raw error messages from Convex, Chargily, or Clerk
- Stack traces
- Internal IDs or technical jargon

## Accessibility

### Keyboard Navigation
- All interactive elements reachable via Tab
- Frosted UI components handle focus trapping natively
- Escape closes modals and dropdowns (handled by Frosted UI)
- Tab navigation bar: arrow keys navigate between tabs (Frosted UI `Tabs` handles this)

### ARIA
- Frosted UI components handle ARIA attributes automatically (Radix UI primitives)
- Icon-only buttons have `aria-label`
- Live regions (`aria-live="polite"`) for dynamically updated content (feed new posts indicator, leaderboard updates)

### Color Contrast
- All text meets WCAG AA contrast ratio on dark backgrounds
- Error and success states never rely on color alone — always paired with an icon or text label

### Mobile
- Minimum tap target size: 44×44px for all interactive elements
- The entire community SPA is fully usable on a 375px-wide screen

## Inline Editing (Owner About Tab)

- Hover state on editable areas: subtle `bg-elevated` highlight + pencil icon appears
- Click to enter edit mode: Frosted UI `TextInput` or `TextArea` replaces the rendered content
- Auto-save on blur: mutation fires when focus leaves the field
- Success feedback: field returns to rendered state — no toast for routine saves
- Error on save failure: Frosted UI `TextInput error="..."` inline error message + field stays in edit mode

## Spacing Adjustments (Frosted UI Scale)

When migrating from the old spacing system to Frosted UI's approach:

| Old pattern | Frosted UI equivalent | Notes |
|---|---|---|
| `p-6` modal padding | Frosted UI `Inset` or `Dialog.Content` default padding | Let `Dialog` handle it |
| `p-4` card padding | Frosted UI `Card` default padding | Built into `Card` component |
| `px-6 py-3` buttons | Frosted UI `Button` default padding | Built into `Button` |
| `gap-4` between items | Frosted UI `Flex gap="4"` | Use `Flex` component |
| `gap-6` between sections | `gap-6` still valid (standard Tailwind) | No change needed |
| `p-5` inside cards | `p-5` (Tailwind) | Still valid |

Frosted UI `Flex`, `Box`, `Inset`, `Grid` all have their own spacing props that map to the
Tailwind scale. Prefer using these components over raw Tailwind gap/padding utilities for
component-internal layout.

## Glass / Blur in UX

Frosted UI's glass surfaces add depth to the following UI elements:

| Element                              | Blur level                                     | Implementation                                |
| ------------------------------------ | ---------------------------------------------- | --------------------------------------------- |
| Modal overlay backdrop               | `backdrop-blur-none`                           | Frosted UI `Dialog`                           |
| Dropdown menus                       | `backdrop-blur-xl`                             | Frosted UI `Popover` / `DropdownMenu` default |
| Hover highlights on interactive rows | `bg-mauve-4/60 backdrop-blur-sm`               | Custom — see DESIGN_SYSTEM.md                 |
| Floating tooltips                    | `backdrop-blur-md`                             | Frosted UI `Tooltip` default                  |
| Top bar (sticky)                     | Optional: `backdrop-blur-md` + `bg-surface/80` | For scroll-linked depth effect                |
