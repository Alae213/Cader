## Direction & Tone

**Cader feels:** Focused. Purposeful. Premium-dark.

This is not a flashy Western SaaS product. It is infrastructure for Algerian knowledge
creators. The Frosted UI visual language gives us a sophisticated, glass-layered dark
interface — confident, crisp, and community-native. 

**Tone words:** Crisp. Glass-dark. Intentional. Modern. Community-native.
**Avoid:** pastel softness, 
generic SaaS aesthetics, light mode.

## Colour Palette

> All tokens defined in `tailwind.config.ts` under the `colors` key.
> Dark mode only. Token names are stable — values can change for light mode without touching components.
> Uses Frosted UI's 12-step Radix color scale (step-1 through step-12, NOT 50–950).
> With the `frostedThemePlugin()` added, standard Tailwind color steps (e.g. `blue-200`)
> do NOT work — always use `blue-1` through `blue-12`.

### Dark Surfaces (Neutral/Mauve Scale)

The surface hierarchy is the foundation of Cader's layout. Every UI layer uses this stack.

| Token         | Value                | Use                                            |
| ------------- | -------------------- | ---------------------------------------------- |
| `bg-canvas`   | `#111111`            | Page root — the deepest layer                  |
| `bg-base`     | `#222222`            | Default page background                        |
| `bg-surface`  | `#2A2A2A`            | Cards, panels, sidebars                        |
| `bg-elevated` | `#313131`            | Hover states, dropdowns, popovers              |
| `bg-overlay`  | `#484848`            | Modal surfaces, floating panels                |
| `bg-glass`    | `rgba(42,42,42,0.8)` | Frosted glass panels (with `backdrop-blur-xl`) |

> These map to mauve/neutral steps 1–12. When defining the Tailwind config, the
> canvas/background sits at the very bottom (darkest) and overlay at the top (lightest
> surface before content reaches pure white text).

### Frosted UI Heading & Text Components

Prefer these over raw HTML elements:

```tsx

// Card title

// Body paragraph
```

### Letter Spacing

| Class        | Value    | Use                        |
| ------------ | -------- | -------------------------- |
| `tracking-0` | -0.05em  | Display headings           |
| `tracking-1` | -0.035em | Titles, subtitles          |
| `tracking-2` | 0        | Body text (default)        |
| `tracking-3` | 0.025em  | All-caps labels, badges    |
| `tracking-4` | 0.05em   | Wide tracking for emphasis |

### Rules
- Line height for body: `leading-3` (16px) or `leading-4` (18px)
- Line height for headings: `leading-tight` (`leading-6` or tighter) — Frosted UI handles this via `tight` variant
- Letter spacing: `-0.025em` on titles and display text (tight, intentional)
- Never use font weights below 400 in dark mode (legibility)
- Use `tracking-3` on all-caps badges and category labels

## Shadows & Glass / Blur

Dark mode shadows use Frosted UI's glass/blur approach — no hard drop shadows.

### Elevation via Surface Stack (preferred over shadows)

Cader uses the surface hierarchy (`bg-surface` → `bg-elevated` → `bg-overlay`) for elevation.
This is cleaner than box-shadows in dark mode and matches Frosted UI's philosophy.

### never use Box Shadows - no shadow

### Frosted Glass Effect

Use Frosted UI's glass utilities for floating panels, modals, and overlays:

```tsx
// Frosted glass overlay panel
<div className="bg-overlay/80 backdrop-blur-xl border border-mauve-4">
  Content with glass effect
</div>

// Lighter glass for hover/active states
<div className="bg-mauve-4/60 backdrop-blur-md">
  Subtle glass highlight
</div>
```

## Dark Mode Only

Cader v1 is **dark mode only**. No `dark:` Tailwind classes are needed — all tokens are
dark-mode values by default.

### How It Works

Frosted UI's `WhopApp` component with `appearance="dark"` sets the entire tree to dark.
All Cader components are designed for dark backgrounds. When light mode is added in v1.1,
the Frosted UI `appearance` prop will be switched to `"inherit"` and semantic tokens will
be remapped to light-mode values.

```tsx
// app/layout.tsx
import { WhopApp } from "@whop/react";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <WhopApp appearance="dark">
          {children}
        </WhopApp>
      </body>
    </html>
  );
}
```

## What to Avoid

- Light backgrounds anywhere in the UI
- Multiple accent colors competing (accent = Chargily green only)
- Animations on every interaction (Framer Motion only for meaningful transitions)
- Rounded corners above `rounded-2xl` on content cards
- Standard Tailwind color steps (`blue-500`, `gray-200`) — always use Frosted UI scale (`blue-6`, `mauve-4`)
- Hardcoded hex values anywhere in component files — always use token names
- Using raw HTML typography elements instead of Frosted UI `Heading`/`Text` components "
