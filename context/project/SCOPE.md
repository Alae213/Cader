## In Scope — v1 (MVP)

These are the things Cader must do on day one to be a usable product.

**Platform**
- [ ] Public landing page (`/`) with CTA, hero, and Clerk auth modal trigger
- [ ] Static help page (`/help`)
- [ ] Three routes only: `/`, `/[communitySlug]`, `/help`

**Community**
- [ ] Community creation modal (2-step: name/slug + pricing/Chargily/wilaya)
- [ ] Community slug — auto-generated from name, editable, real-time uniqueness validation
- [ ] About tab — public, no auth required (VSL embed, description, links, member stats, join button)
- [ ] Inline editing of About tab content for owners (click-to-edit, auto-save on blur)
- [ ] Community pricing: monthly / annual / one-time / free — set in DZD
- [ ] Platform free tier: 50 lifetime unique members before creator must subscribe
- [ ] Platform subscription: 2,000 DZD/month — unlocks unlimited new members
- [ ] Community locked tier: no new joins when free limit is hit, existing members retain access

**Authentication**
- [ ] Clerk auth modal (sign-up / sign-in) — email magic links + social
- [ ] Clerk webhook → Convex users table sync on `user.created`
- [ ] Join intent preserved through auth flow (EC-1)
- [ ] Top bar and tab navigation hidden for unauthenticated visitors and non-members (EC-11)

**Onboarding**
- [ ] Onboarding modal: full name + phone number
- [ ] Paid community: Chargily checkout step in onboarding; membership granted via webhook only (EC-3)
- [ ] Free community: access granted immediately on name + phone submit
- [ ] Partial onboarding creates no membership record (EC-2)
- [ ] Graceful error if Chargily keys missing (EC-4)

**Payments — Chargily Pay**
- [ ] Creator stores Chargily Public + Secret Key in community settings (encrypted at rest)
- [ ] Server-side checkout session creation via Chargily API
- [ ] Webhook endpoint `/api/webhooks/chargily` — `checkout.paid`, `checkout.failed`, `checkout.canceled`
- [ ] Signature verification on every webhook request
- [ ] Membership granted based on webhook state only (never optimistic)
- [ ] **Manual monthly rebilling:** Daily cron job detects expiring memberships, sends renewal email, sets inactive after grace period (EC-9)
- [ ] Per-classroom payment (one-time fee via Chargily)

**Community Feed**
- [ ] Post composer: text, image, video embed (YouTube/Vimeo), GIF, poll
- [ ] Post categories (owner defines, members filter)
- [ ] Threaded comments
- [ ] @mentions
- [ ] Post upvotes (awards points)
- [ ] Pin post (owner/admin only)
- [ ] Delete post / comment (owner/admin only)
- [ ] Add post modal + open post modal (click on post)

**Classrooms**
- [ ] Classroom grid (thumbnail + title + progress bar)
- [ ] Create classroom (owner: + card in grid)
- [ ] Module / page tree in left sidebar (modules as folders, pages as lessons)
- [ ] Lesson content: rich text + video embeds + file attachments
- [ ] Lesson content editing via `/` command (owner only)
- [ ] Progress tracking per page viewed
- [ ] Classroom access model: open / level-gated / price-gated / level+price
- [ ] Level gate evaluated before payment gate — always (EC-6)
- [ ] Access gate overlay for locked classrooms (level message or payment CTA)
- [ ] "Start Here" onboarding course pattern (optional, owner-configured)
- [ ] Classroom viewer: left sidebar (tree) + right content panel

**Leaderboard + Gamification**
- [ ] 5-level progression system
- [ ] Points from post upvotes (append-only `pointEvents` table — ADR-008)
- [ ] Leaderboard: Top members — Last 7 days / Last 30 days / All time
- [ ] Level visualization (5-level progress bar/indicator)
- [ ] Level is always derived from `pointEvents` sum — never stored

**Modals**
- [ ] Profile modal (avatar, full name, contribution activity map, communities joined, communities created)
- [ ] Settings modal (profile, notifications, admins, billing, danger zone)
- [ ] Create new community modal (same 2-step as on `/`)
- [ ] Explore modal (lightweight community directory — search/browse)
- [ ] Onboarding modal (per flow above)
- [ ] Classroom viewer (within `/[communitySlug]` — no route change)

**Settings**
- [ ] Profile: display name, avatar, bio
- [ ] Notifications: email/in-app toggle (basic)
- [ ] Admins: add / remove community admins
- [ ] Billing: view subscription status, renew, cancel
- [ ] Danger zone: delete community (blocked if active paying members — EC-7)
- [ ] Account: sign out, delete account
- [ ] Minimum 1 admin enforced — last admin cannot self-remove (EC-8)

**Tab Persistence**
- [ ] Tab state stored in localStorage keyed by community slug
- [ ] localStorage is a UI hint only — access always re-validated server-side (EC-13)
- [ ] Default tab: About (unauthenticated) or Community/feed (member)

## Out of Scope — v1 (Possible Later)

These are deliberate deferrals — not forgotten, just not v1.

| Feature | Why deferred |
|---|---|
| Arabic / French language support | i18n complexity; target tech-savvy creators use English/French professionally |
| Light mode | Dark mode only simplifies CSS token system for MVP (see ADR-006) |
| Full analytics dashboard | Placeholder removed from v1 — real data requires more instrumentation |
| Creator revenue analytics (payouts, earnings overview) | Chargily handles this natively in creator's own dashboard |
| Native video upload / hosting | Expensive infrastructure; embed URLs cover the need (see ADR-007) |
| Mobile apps (iOS / Android) | Web-first MVP; mobile-responsive web covers the learner persona |
| Direct messaging between members | High complexity, not core to v1 value prop |
| Email digest / newsletter | Notifications are in-app and basic email for v1 |
| Affiliate / referral system | Post-PMF growth feature |
| Custom domain for communities | Possible v2 — slug-based URLs cover v1 |
| Multi-language content in courses | Tied to i18n deferral |
| API / embed widgets | For third-party integrations — v2 |
| Creator onboarding wizard (guided setup) | Nice to have; basic modal is sufficient for v1 |
| Attendance / live events | Async learning only for v1 |

## Never In Scope

These are off the table entirely, not just deferred.

| Feature | Reason |
|---|---|
| Stripe / PayPal integration | Does not support Algerian CIB/Edahabia — structurally incompatible with the market |
| Multi-currency support | DZD only — by design (see ADR-003) |
| Platform taking a cut of community revenue | Creator owns their Chargily account — platform subscription is the only revenue model |
| Platform-level Creator / Student roles | Roles are community-scoped only (see ADR-004) |
| URL changes on tab switch | SPA with localStorage tab state only — URLs are fixed by design |

## Open Questions

- [ ] What point values are assigned per action? (upvote = 1 point? are there other earning actions beyond upvotes?)
- [ ] What are the exact DZD thresholds for level 1–5? (e.g. 0→100→500→1000→5000 points?)
- [ ] Will the Explore modal show all public communities or only specific ones the owner marks as discoverable?
- [ ] Are community avatars uploaded to Convex file storage, or are they external URLs?
- [ ] What is the exact data shown in the "contribution activity map" on the Profile modal? (post count? login streak? upvotes given?)

### Email Infrastructure
- [ ] What email service? Resend? SendGrid? Or skip custom emails for v1 and rely on Chargily receipts + magic links only?

### Content Moderation
- [ ] Do we need a basic report system (flag post/comment/user) in v1, or is "admin delete" enough for launch?

### Member Roles & Permissions
- [ ] Permission model — owner + admins only, or a third role (moderator)? Who can: create classrooms, pin posts, manage categories, view payment history?

### File/Media Storage
- [ ] Where do uploaded images and lesson attachments go? Convex file storage? UploadThing? S3? What max file size?

### In-App Notifications
- [ ] Does v1 need a notification bell with in-app notifications (new comment, @mention, upvote, payment), or just @mention inline + Chargily flow?

### Post Images
- [ ] Single image per post, or multi-image gallery? Max images? Preview/caption?

### GIF Support
- [ ] How do users add GIFs? GIPHY API picker or paste URL?

### Polls
- [ ] Poll rules — max options? Expiry duration? Can voters see results before voting? Can owner edit after posting?

### Classroom Access on Expiry
- [ ] When membership expires (EC-9), immediate lockout of level-gated classrooms or grace period?

### Help Page
- [ ] What content goes on `/help`? FAQ? How-to? Payment troubleshooting? Is copy ready or do we write it?

### Search
- [ ] Any search in v1 beyond Explore modal? Post search? Member search?

### Reserved Slugs
- [ ] Block slugs like `explore`, `help`, `api`, `settings` to avoid route conflicts?

### PWA / Offline
- [ ] Any offline/PWA support for v1 or v1.1? Or purely online-only?

### Input Validation & Limits
- [ ] Character limits for: community name, description, post text, comment text, classroom/lesson titles? Reasonable defaults or specific limits?

### Webhook Retry Handling
- [ ] Need a webhook retry log / dead-letter queue, or simple retry handling enough?

### Account Deletion
- [ ] On account delete: hard delete (remove everything) or soft delete (anonymize but keep content)?
