## Commit Style

**Conventional Commits** format:

```
type(scope): short description

[optional body]
[optional footer — e.g. "Closes T-12"]
```

**Types:**
| Type | When to use |
|---|---|
| `feat` | New feature or behaviour |
| `fix` | Bug fix |
| `chore` | Config, dependencies, tooling (no production code change) |
| `refactor` | Code restructure (no behaviour change) |
| `docs` | Documentation updates (including context/ files) |
| `style` | Formatting only (no logic change) |
| `test` | Adding or updating tests |
| `perf` | Performance improvement |

**Examples:**
```
feat(chargily): add webhook handler for checkout.paid event
fix(onboarding): preserve join intent through Clerk auth redirect (EC-1)
docs(context): update ROADMAP with v1 task statuses
chore(deps): upgrade Next.js to 14.2.0
```

**Rules:**
- Subject line: max 72 characters, imperative mood ("add" not "added")
- Reference task number in footer when applicable: `Closes T-12` or `Part of T-8`
- Never commit `.env.local` or any file containing real secrets — verify with `git status` before every commit

## Local Development

### Prerequisites
- Node.js 20+
- npm 10+
- Convex CLI: `npm i -g convex`
- A Convex account and project set up at [convex.dev](https://convex.dev)
- A Clerk account and application at [clerk.com](https://clerk.com)

### Running locally

Terminal 1 — Convex:
```bash
npx convex dev
```

Terminal 2 — Next.js:
```bash
npm run dev
```

App runs at `http://localhost:3000`.

### Webhook testing

Use ngrok or Cloudflare Tunnel to expose localhost for Clerk and Chargily webhooks:
```bash
ngrok http 3000
```
Register the tunnel URL in:
- Clerk dashboard → Webhooks
- Chargily dashboard → Webhooks

## Deploy Process

All commits go to `main`. Vercel auto-deploys `main` on push.

1. Run `npm run build` locally to verify no build errors
2. Commit with conventional commit format
3. Push to `main` — Vercel deploys automatically
4. Verify the Vercel deploy URL before treating it as done

---

## Convex Workflow

- Schema changes go in `convex/schema.ts`. Run `npx convex dev` to apply.
- Function changes hot-reload automatically during `npx convex dev`.
- Run `npx convex deploy` to push functions to production.
- Only push production-ready code — Convex prod is shared with the live app.

## Pre-commit Checklist

Before every commit:

- [ ] `npm run build` passes with no errors
- [ ] No `console.log` left in production paths (Convex functions, Route Handlers)
- [ ] No hardcoded secrets, API keys, or credentials in any file
- [ ] `.env.local` is NOT staged (`git status` check)
- [ ] Conventional commit message format used
- [ ] Task status updated in TASK-LIST.md if applicable
