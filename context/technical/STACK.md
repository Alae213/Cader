## Framework

**Next.js 14+ (App Router)**

The App Router gives us server components, streaming, and file-based routing out of the box.
The `/[communitySlug]` route is a single-page application — the URL never changes, but the
App Router's nested layout system handles the persistent shell cleanly.

Key patterns used:
- Server Components for all data fetching and initial render
- Client Components for interactive UI (feed, classroom viewer, modals)
- Route Handlers (`app/api/`) for webhooks only (Clerk, Chargily)
- Middleware for Clerk auth + membership gating

## UI Layer

**React** (comes with Next.js)
Component layer. Client Components are marked `"use client"` explicitly.

**Tailwind CSS**
Utility-first styling. All design tokens (colors, spacing, typography, border radius) are
defined in `tailwind.config.ts`. No inline styles. No CSS modules except where Tailwind
cannot reach.

**shadcn/ui**
Accessible base components: dialogs, dropdowns, sheets, forms, toasts.
We own the code — components live in `components/ui/`. Never import from the registry
after initial install; modify in place.

**Framer Motion (`motion`)**
Complex animations and micro-interactions. Used sparingly — only for: modal open/close,
tab transition, classroom progress, and the Wilaya map hover states.

## Backend & Database

**Convex**

Real-time reactive database, serverless functions, and file storage — all in one.

| Convex concept | What we use it for |
|---|---|
| `query` | Read data (feed posts, membership status, classroom progress) |
| `mutation` | Write data (create post, mark lesson complete, provision membership) |
| `action` | External API calls (Chargily checkout creation, webhook verification) |
| `useQuery` | Live-reactive UI (feed, leaderboard, points — update without polling) |
| `useMutation` | Client-triggered writes |
| File storage | Avatars, lesson attachments (not video — see ADR-007) |

High-traffic indexes defined on: `communityId`, `userId`, `lessonId`, `membershipStatus`.

All backend logic lives in `convex/`. There is no separate API server.

## Authentication

**Clerk**

Handles the full auth lifecycle: sign-up, sign-in, session management, JWT issuance.

- **No email/password.** Clerk provides email magic links and social sign-in.
- **Clerk → Convex sync:** A Clerk webhook (`/api/webhooks/clerk`) fires on `user.created`
  and writes a new row to the Convex `users` table. Clerk `userId` is the foreign key throughout.
- **Middleware:** Clerk middleware runs globally. A secondary custom layer checks active
  `memberships` records before granting access to feed/classroom/map tabs.
- **Auth modal:** Clerk's hosted component is triggered client-side (on CTA click or Join click).
  It is not a full-page redirect.

## Payments

**Chargily Pay**

The only payment gateway that accepts Algerian CIB and Edahabia bank cards natively.
This is the entire commercial reason this product exists.

Integration model:
- Creator stores their own Chargily **Public Key** and **Secret Key** in their community
  settings. Keys are encrypted at rest in Convex.
- Checkout flow: server-side session creation via Chargily API → redirect to Chargily hosted
  checkout page → webhook fires on payment success.
- Webhook endpoint: `/api/webhooks/chargily`
  Listens for: `checkout.paid`, `checkout.failed`, `checkout.canceled`
  Signature verified on every request before any action is taken.
- **Note:** Chargily has no native subscription API. Manual monthly rebilling is implemented
  via a daily Convex scheduled action that detects expiring memberships and sends renewal emails.
- Currency: **DZD only.** No conversion. No other currencies. Ever.

Platform financial model:
- Creator's Chargily account receives all community and classroom revenue directly.
- The platform charges creators a separate subscription fee (platform billing — separate flow).
- Platform never holds or moves creator funds.

| Prisma / Drizzle | No separate ORM needed — Convex handles schema and queries |
| React Native / Expo | Web only for MVP |
| Socket.io / Pusher | Convex live queries handle all real-time needs |
| Mux / Cloudflare Stream | No native video hosting — embed URLs only (see ADR-007) |
| i18n (next-intl, etc.) | English only for MVP (see ADR-005) |

## Rationale Summary

Every choice above was made against three constraints:

1. **Free tiers first** — solo builder with no infrastructure runway. Convex, Clerk, and Vercel all run on generous free tiers.
2. **Real-time by default** — feed, leaderboard, and lesson progress must update live. Convex `useQuery` handles this without polling.
3. **Algerian market fit** — Chargily Pay is the non-negotiable foundation. Everything else serves the platform it enables.
